## Основные команды Git

* *git init – инициализация локального репозитория*
* *git config --global user.name "Ваше имя английскими буквами" - задать имя пользователя
* *git config --global user.email ваша почта@example.com
* *git status – получить информацию от git о его текущем состоянии* - задать почту пользователя
* *git add название файла(пишешь начало потом tab) – добавить файл или файлы к следующему коммиту*
* *git add . - добавить все файлы в репозитории к комиту*
* *git commit -am "message" - объединяет в себе 2 комманды "git add и git commit -m" не работает когда сознан новый файл*
* *git commit -m “message” – создание коммита.*
* *git commit -a - объединяет в себе 2 комманды "git add и git commit -m" только после ввода команды появится окно куда надо будет ввести сам комит*
* *git commit --amend - изменить только сообщение вашего последнего коммита(после ввода команды, появится окно где изменяем сообщение комита, потом закрываем окно, на этом все)*
* *git log – вывод на экран истории всех коммитов с их хеш-кодами*
* *git log --oneline - вывод на экран истории всех коммитов в одну строку каждый*
* *git log --graph - вывод на экран истории всех коммитов в виде дерева с переходми по веткам*
* *git log -p --all - подробный лог*
* *git log --all --oneline - подробный лог*
* *git log -p - вывод на экран истории всех коммитов подробно*
* *git log --graph --oneline --decorate - история коммитов в виде графика для текущей ветки*
* *git log --all --graph --oneline --decorate - история коммитов по всем веткам*
* *git checkout хеш(либо полностью, либо первые 4 символа) - переход от одного коммита к другому*
* *git checkout название ветки - переход на нужную ветку*
* *git checkout master – вернуться к актуальному состоянию и продолжить работу*
* *git checkout -b имя ветки - объединяет в себе 2 комманды "git branch название ветки и git checkout название ветки", тоесть создаем ветку и сразу в нее переходим*
* *git diff – увидеть разницу между текущим файлом и закоммиченным файлом*
* *gitk --all - запускает оболочку git с commit*
* *git rm название файла - удаление файла из директории MacOS, Linux*
* *git del название файла - удаление файла из директории Windows* 
* *git merge название ветки - слияние главной с выбранной веткой, производится из ветки master, при слиянии если в одинаковых местах но в разных ветках были разные данные то задаст вопрос оставить или заменить или принять оба варианта*
* *git merge origin - слияние удалённого репозитория с локальным выполняется параметром merge с указанием имени удалённого репозитория.*
* *git merge --abort - прервать слияние в случае конфликта, это позволяет остановить процесс слияния и вернуть состояние, с которого этот процесс был начат.*
* *git reset - восстановить конфликтующие файлы до стабильного состояния*
* *git branch - показывает список веток там где (*) на той и находимся*
* *git branch название - создание отдельной ветки*
* *git branch -d название ветки - удаление указанной ветки (когда было слияние)*
* *git branch -D название ветки - принудительное удаление указанной ветки(когда не было слияния)*
* *git branch -a - вывести список удалённых веток*
* *git branch --all - все ветки которые могут не отображаться в локальном репозитории, который скопировали из удаленного репозитория, будут выведены все ветки локальные и удаленные выделенны красным цветом и написаны через / (составные)*
* *ls - показывает все файлы которые есть в репозитории*
* *git clone ссылка - скопировать репозиторий в локальный*
* *git clone ссылка название директории(папки) - скопировать репозиторий в локальный в определенное место*
* *git remote add upstream ссылка на удаленный репозиторий который склонирован - Склонированный репозиторий имеет одну привязку к удалённому репозиторию, названную origin, которая указывает на вашу копию на GitHub, а не на оригинальный репозиторий, чтобы отслеживать изменения и в нём, вам нужно будет добавить другую привязку, названную, например, upstream.*
* *git push ссылка - выложить локальный репозиторий в git hub* 
* *git push - синхронизировать из локального в удаленный репозиторий* 
* *git push origin <branch_name> - отправить локальную ветку <branch_name> в репозиторий origin*
* *git push origin --delete <branch_name> - удаление ветки <branch_name> из удаленного репозитория github origin*
* *git push origin :<branch_name> - удаление ветки <branch_name> из удаленного репозитория github origin*
* *git push -u origin new_branch - передать новую ветку в удалённый репозиторий можно параметром push с флагом -u, указав имя репозитория и имя ветки.*
* *git pull - загрузка с gitgub.com обновлений вместе со сливанием веток, что он выполняет работу «git fetch» ​​и «git merge»*
* *git remote rename <name> - переименовать удаленный репозиторий на <name>
* *git remote remove <name> - удалить удаленный репозиторий с именем <name>*
* *git remote - список настроенных удалённых репозиториев*
* *git remote -v - просмотреть адреса для чтения и записи, привязанные к репозиторию*
* *git remote add <name> <url> - добавить удалённый репозиторий к существующему локальному по ссылке <url> и <name> присвоить ему имя* 
* *git remote show origin -  подробные сведения об удалённом репозитории можно с помощью параметра remote show с указанием имени репозитория — например, origin, отображает список веток, связанных с удалённым репозиторием, а также рабочих станций, подключённых для получения и отправки файлов.*
* *git fetch [remote-name] - для получения данных из удалённых проектов*
* *Чтобы Git не отслеживал не нужные файлы в директории надо создать файл *.gitignore* и туда добавить название файлов*
* *Когда делаешь fork, при его создании будет предложено скопировать все ветки или только основную*
* *Когда клонируешь со своего удаленного репозитория в свой локальный скорее всего будет только одна основная ветка, чтобы посмотреть git branch --all, красным будут гореть ветки которые можно будет добавить git checkout (branch_name)*
* *Чтобы закачать изменения с удаленного репозитория(с которого делали fork) в свой удаленный репозиторий, нужно зайти в свой удаленный репозиторий(fork) и нажать кнопку <Synk fork>, потом <Update branch> и потом залить к себе на локальный репозиторий git pull*

## Чтобы добавлять несколько папок в удаленный репозиторий
##### 1.Нужно создать репозиторий(папку)
##### 2.Внутри созданного репозитория (папки) создать нужное колличество папок
##### 3.Нажать правой кнопкой на репозитории - открыть во встроенном терминале
##### 4.Вбить команду git add <название нужной папки>
##### 5.Вбить команду git commit -m <"написать комит">
##### 6.Желательно в каждой папке сделать .gitignore(скопировать 398 строк) чтобы в удаленном репозитории не было не нужных файлов

#### Fork - скопировать удаленный репозиторий к себе в удаленный репозиторий

## Последовательность выполнения Pull Requests(не рекомендуется делать из ветки main, так как задвоится комит на удаленном репоитории не своем)
##### 1.Fork(копия чьего то удаленного репозитория к себе в удаленный на github)
##### 2.Clone url (делаем клон удаленного в локальный репозиторий свой)  
##### 3.Вносим изменения (потом добавляем и делаем комит)
##### 4.Push(заливаем из локального в удаленный свой)
##### 5.Переходим в удаленный репозиторий с которого нужно сделать pull requests(отправка своего варианта (изменения) создателю от кого копировали)
##### 6.Нажимаем new pull requests(проверяем из какой ветки моего удаленного репозитория в какую ветку удаленного репозитория будет происходить)
##### 7.Нажимаем Create pull requests (делаем комит) 
##### 8.Нажимаем Create pull requests

## Последовательность выполнения правильного Pull Requests
##### 1.Fork(копия чьего то удаленного репозитория к себе в удаленный на github)
##### 2.Clone url (делаем клон своего удаленного(fork) в свой локальный репозиторий)
##### 3.Cделать отдельную ветку.
##### 4.Вносим изменения (потом добавляем и делаем комит)
##### 5.Push(заливаем из локального в удаленный свой) выйдет ошибка, ввести предложенную команду
##### 6.Перейти в свой удаленный репозиторий(fork) на github.
##### 7.Появится кнопка Compare&pull request, нажать и следовать дальше.

## Подсказка по cmd

Команда смены директории
```sh
cd "c:\folder_name"
```
Команда смены директории (возврат на шаг назад в пред. директорию)
```sh
cd ..
```
Команда смены директории (когда в репозитории есть папки)
```sh
cd имя папки
```
Команда отображения текущей директории Linux, MacOs
```sh
pwd
```
Листинг текущей директории Windows
```sh
dir
```
Листинг текущей директории Linux, MacOs
```sh
ls
```
Для удаления файла WacOs, Linux
```sh
rm  <file name>
```
Для удаления файла Windows
```sh
del  <file name>
```
